                <div class="response-text" vid="65">KV-Cache (Key-Value Cache) is a crucial optimization for autoregressive model inference. 

In a standard transformer, generating each new token requires recomputing the attention keys (K) and values (V) for all previous tokens in the sequence. This leads to redundant $O(N^2)$ computations as the sequence grows.

By storing (caching) the K and V tensors for processed tokens, the model only needs to compute K and V for the *newly* generated token. This reduces the per-step complexity to $O(N)$, significantly improving throughput and reducing latency during generation.

Modern implementations like PagedAttention (vLLM) further optimize this by managing the cache in non-contiguous memory blocks, similar to virtual memory in operating systems.</div>
